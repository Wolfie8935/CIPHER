# TASK 3 — File Corruption & Ransomware Mechanic: RED's Second Objective

> **Priority**: 🟡 HIGH — Adds strategic depth + judges love multi-objective agents  
> **Effort**: 6–8 hours  
> **Dependencies**: None (parallel-safe)

---

## OBJECTIVE

Currently RED has one objective: **exfiltrate** the classified file from zone 3. This task adds a second full objective: **file corruption**. RED can corrupt, encrypt, or ransomware files across the network — destroying BLUE's data integrity. BLUE gets corresponding **antidote, backup, and integrity checking** countermeasures.

This creates a genuine **strategic fork** that agents must reason about: _"Do I exfiltrate for max reward, corrupt to deny BLUE forensic evidence, or do BOTH for a combo bonus?"_ This is exactly the kind of multi-objective planning that impresses judges.

## DESIGN: THE FULL CORRUPTION SYSTEM

### 3.1 — RED Corruption Actions (3 new ActionTypes)

#### `CORRUPT_FILE` — Targeted File Corruption
| Property | Value |
|----------|-------|
| **ActionType** | `corrupt_file` |
| **Valid zones** | All zones (0-3) |
| **Target** | Specific file on current node |
| **Suspicion cost** | +0.08 |
| **Effect** | File is marked corrupted; cannot be used by BLUE forensics for path reconstruction |
| **Reward** | +0.12 per file corrupted |
| **Counter** | BLUE's `deploy_antidote` can restore it |
| **Limit** | Max 5 files per episode |

**How it works in the episode runner:**
```python
# When RED chooses corrupt_file on node N:
node_files = graph.nodes[state.red_current_node].get("files", [])
if not node_files:
    return ActionExecutionResult(success=False, reason="no_files_on_node", state_delta={})

target_file = action.target_file or node_files[0]
if target_file not in node_files:
    return ActionExecutionResult(success=False, reason="file_not_found", state_delta={})

# Mark as corrupted
if state.red_current_node not in state.corrupted_files:
    state.corrupted_files[state.red_current_node] = []
state.corrupted_files[state.red_current_node].append(target_file)
state.corruption_count += 1
state.red_suspicion_score = min(1.0, state.red_suspicion_score + 0.08)

# Log
state.episode_log.append({
    "step": current_step,
    "agent_id": action.agent_id,
    "action_type": "corrupt_file",
    "payload": {"target_file": target_file, "node": state.red_current_node},
    "result": {"success": True, "total_corrupted": state.corruption_count},
})
```

#### `RANSOMWARE_DEPLOY` — Mass Encryption
| Property | Value |
|----------|-------|
| **ActionType** | `ransomware_deploy` |
| **Valid zones** | Zone 2+ only (Sensitive, Critical) |
| **Target** | Current node — ALL files encrypted |
| **Suspicion cost** | +0.20 (very high — ransomware is noisy) |
| **Effect** | Every file on node is corrupted; node becomes "locked" for 3 steps |
| **Reward** | +0.08 per file × number of files (can be very high on file-heavy nodes) |
| **Counter** | BLUE's `backup_snapshot` prevents it; `deploy_antidote` restores 1 file at a time |
| **Limit** | 2 ransomware deployments per episode |
| **Special** | If node has a BLUE backup_snapshot, ransomware FAILS and triggers an alarm (+0.15 suspicion) |

**How it works:**
```python
# When RED chooses ransomware_deploy:
zone = graph.nodes[state.red_current_node].get("zone")
zone_val = zone.value if hasattr(zone, "value") else int(zone)

if zone_val < 2:
    return ActionExecutionResult(success=False, reason="zone_too_low", state_delta={})

if state.ransomware_deploys >= 2:
    return ActionExecutionResult(success=False, reason="ransomware_limit_reached", state_delta={})

# Check for BLUE backup
if state.red_current_node in state.backup_nodes:
    state.red_suspicion_score = min(1.0, state.red_suspicion_score + 0.15)
    return ActionExecutionResult(success=False, reason="backup_blocked", state_delta={
        "alarm_triggered": True,
        "message": "Ransomware blocked by BLUE backup — alarm triggered"
    })

# Encrypt all files
node_files = list(graph.nodes[state.red_current_node].get("files", []))
state.corrupted_files[state.red_current_node] = node_files
state.corruption_count += len(node_files)
state.ransomware_deploys += 1
state.red_suspicion_score = min(1.0, state.red_suspicion_score + 0.20)

# Lock node for 3 steps
state.locked_nodes[state.red_current_node] = 3
```

#### `SILENT_CORRUPT` — Stealthy Corruption
| Property | Value |
|----------|-------|
| **ActionType** | `silent_corrupt` |
| **Prerequisite** | Must have `read_file` on the same file in a previous step |
| **Valid zones** | All |
| **Suspicion cost** | +0.02 (nearly invisible) |
| **Effect** | File corrupted silently; BLUE forensics sees it as normal for 3 more steps |
| **Reward** | +0.10 per file |
| **Special** | Corruption is "hidden" — BLUE's `integrity_check` won't detect it for 3 steps |

```python
# When RED chooses silent_corrupt:
target_file = action.target_file
if target_file not in state.red_read_files:
    return ActionExecutionResult(success=False, reason="must_read_first", state_delta={})

# Silent corruption — delayed visibility
state.pending_corruptions.append({
    "node": state.red_current_node,
    "file": target_file,
    "visible_at_step": current_step + 3,  # Hidden for 3 steps
})
state.corruption_count += 1
state.red_suspicion_score = min(1.0, state.red_suspicion_score + 0.02)
```

### 3.2 — BLUE Antidote Actions (3 new ActionTypes)

#### `DEPLOY_ANTIDOTE` — File Restoration
| Property | Value |
|----------|-------|
| **ActionType** | `deploy_antidote` |
| **Target** | Specific node |
| **Effect** | Restores 1 corrupted file on target node; applies 3-step corruption immunity |
| **Detection boost** | +0.05 |
| **Limit** | 4 antidotes per episode |

```python
# When BLUE chooses deploy_antidote:
target = action.target_node
if target not in state.corrupted_files or not state.corrupted_files[target]:
    # Antidote on clean node — still sets protection
    state.antidote_protection[target] = 3
    return ActionExecutionResult(success=True, reason="protection_set", state_delta={})

# Restore one file
restored_file = state.corrupted_files[target].pop(0)
state.antidote_count += 1
state.antidote_protection[target] = 3  # 3-step immunity
state.blue_detection_confidence = min(1.0, state.blue_detection_confidence + 0.05)
```

#### `BACKUP_SNAPSHOT` — Preventive Backup
| Property | Value |
|----------|-------|
| **ActionType** | `backup_snapshot` |
| **Valid nodes** | Zone 2+ only (protect high-value assets) |
| **Effect** | Creates an immutable backup; if files are corrupted later, auto-restores next step |
| **Detection boost** | +0.03 |
| **Limit** | 2 backups per episode |
| **Special** | Blocks `ransomware_deploy` entirely; triggers alarm if RED tries |

```python
# When BLUE chooses backup_snapshot:
target = action.target_node
zone = graph.nodes[target].get("zone")
zone_val = zone.value if hasattr(zone, "value") else int(zone)

if zone_val < 2:
    return ActionExecutionResult(success=False, reason="zone_too_low", state_delta={})

state.backup_nodes.add(target)
state.backup_count += 1
state.blue_detection_confidence = min(1.0, state.blue_detection_confidence + 0.03)

# Store current file snapshot for auto-restore
state.backup_snapshots[target] = list(graph.nodes[target].get("files", []))
```

#### `INTEGRITY_CHECK` — File Health Scan
| Property | Value |
|----------|-------|
| **ActionType** | `integrity_check` |
| **Target** | Any node |
| **Effect** | Reveals which files are corrupted (except silent_corrupt within 3-step window) |
| **Detection boost** | +0.08 |
| **Cost** | Uses 1 step (no other action possible) |

```python
# When BLUE chooses integrity_check:
target = action.target_node
corrupted = state.corrupted_files.get(target, [])

# Filter out hidden silent corruptions
visible_corrupted = [f for f in corrupted
                     if f not in [p["file"] for p in state.pending_corruptions
                                   if p["visible_at_step"] > current_step]]

state.blue_detection_confidence = min(1.0, state.blue_detection_confidence + 0.08)

# Add to BLUE's next observation
state.blue_integrity_reports.append({
    "node": target,
    "corrupted_files": visible_corrupted,
    "total_files": len(graph.nodes[target].get("files", [])),
    "scan_step": current_step,
})
```

### 3.4 — Reward Design

#### RED Reward Changes (`cipher/rewards/red_reward.py`)

Add to `RedRewardComponents`:
```python
corruption_score: float = 0.0        # Base corruption reward
corruption_combo_bonus: float = 0.0   # Bonus for exfil + corrupt combo
ransomware_bonus: float = 0.0         # Extra for mass encryption
```

Computation in `compute_red_reward()`:
```python
# Corruption base: 0.12 per corrupted file, capped at 0.60
corruption_score = min(0.60, state.corruption_count * 0.12)

# Combo bonus: if RED BOTH exfiltrated AND corrupted, 1.5x multiplier on corruption
corruption_combo_bonus = 0.0
if exfil_completeness > 0.5 and corruption_score > 0.1:
    corruption_combo_bonus = corruption_score * 0.5  # 50% bonus

# Ransomware bonus: 0.05 per successful ransomware deploy
ransomware_bonus = state.ransomware_deploys * 0.05

# Add to total (after existing computation):
total += corruption_score + corruption_combo_bonus + ransomware_bonus
```

#### BLUE Reward Changes (`cipher/rewards/blue_reward.py`)

Add to `BlueRewardComponents`:
```python
antidote_effectiveness: float = 0.0   # Reward for successful antidotes
backup_saves: float = 0.0             # Reward for backups that blocked ransomware
integrity_detection: float = 0.0      # Reward for finding corrupted files
```

Computation in `compute_blue_reward()`:
```python
# Antidote effectiveness: ratio of restored vs total corrupted
if state.corruption_count > 0:
    antidote_rate = state.antidote_count / state.corruption_count
    antidote_effectiveness = min(0.25, antidote_rate * 0.25)

# Backup saves: 0.15 for each ransomware blocked by backup
backup_blocks = sum(1 for entry in state.episode_log
                    if entry.get("result", {}).get("reason") == "backup_blocked")
backup_saves = backup_blocks * 0.15

# Integrity detection: 0.05 per integrity check that found corruption
integrity_finds = sum(1 for report in state.blue_integrity_reports
                      if report.get("corrupted_files"))
integrity_detection = min(0.20, integrity_finds * 0.05)

total += antidote_effectiveness + backup_saves + integrity_detection
```

### 3.5 — Per-Step Mechanics

Add to the **step processing** in `_episode_runner.py`:

```python
# At the START of each step, process pending mechanics:

# 1. Decrement antidote protection timers
for node in list(state.antidote_protection.keys()):
    state.antidote_protection[node] -= 1
    if state.antidote_protection[node] <= 0:
        del state.antidote_protection[node]

# 2. Decrement locked node timers
for node in list(state.locked_nodes.keys()):
    state.locked_nodes[node] -= 1
    if state.locked_nodes[node] <= 0:
        del state.locked_nodes[node]

# 3. Reveal pending silent corruptions
for pending in list(state.pending_corruptions):
    if current_step >= pending["visible_at_step"]:
        node = pending["node"]
        if node not in state.corrupted_files:
            state.corrupted_files[node] = []
        state.corrupted_files[node].append(pending["file"])
        state.pending_corruptions.remove(pending)

# 4. Auto-restore from backups
for node in state.backup_nodes:
    if node in state.corrupted_files and state.corrupted_files[node]:
        # Backup auto-restores — clear corruption
        state.corrupted_files[node] = []
        state.episode_log.append({
            "step": current_step,
            "agent_id": "system",
            "action_type": "backup_auto_restore",
            "payload": {"node": node},
            "result": {"restored": True},
        })

# 5. Block RED movement into locked nodes
if (action.action_type == ActionType.MOVE and
    action.target_node in state.locked_nodes):
    return ActionExecutionResult(
        success=False,
        reason="node_locked_by_ransomware",
        state_delta={},
    )
```
